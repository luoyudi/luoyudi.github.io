性能相关指标

速率

:::tips
速率即数据率，或者数据传输率或比特率
:::
:::tips
比特 1/0 位
:::
:::tips
连接在计算机网络上的主机在数字信道上传送数据位数的速录。
单位是b/s，kb/s，Mb/s，Gb/s，Tb/s
:::

带宽

:::tips

  1. 带宽原本指某个信号具有的频带宽度，即最高频与最低频之差，单位是赫兹（Hz）
  2. 计算机网络中，带宽用来表示网络的通信线路传送数据之间的能力，通常是指但是时间内从网络中的某一点到零一点所能通过的“最高数据率”。单位是“比特每秒”
    1. 网络设备所支持的最高速度
    2. 单位是b/s，kb/s，Mb/s，Gb/s，Tb/s
      :::

吞吐量

:::tips
表示在单位时间内通过某个网络（或者信道、接口）的数据量
单位是b/s，kb/s，Mb/s，Gb/s，Tb/s
吞吐量收网络的带宽或网络的额定速率的限制
:::

:::tips
数据（报文、分组、比特流）从网络（链路）的一端传送到另一端所需要的时间。也叫延迟或者迟延。
单位是秒
:::
:::tips

  1. 发送时延：从发送分组的第一个比特算起，到最后一个比特结束，发送所需要的时间取决于信道带宽和数据长度
  2. 传播时延：取决于电磁波的传播速度和线路长度
  3. 排队时延：等待输入输出链路可用的时间
  4. 处理时延：检错、找出口
    :::

时延带宽积

:::tips
传播时延x带宽
管道党总此时此刻数据总量
相当于以bit为单位的链路长度
:::

往返时延RTT

:::tips
从发送方发送数据到发送方接收到接收方的确认总共耗时
RTT越大在收到确认之前，可以发送的消息越多
包括

  1. 末端处理时间
  2. 传播延时*2=往返传播时延
    :::

利用率

  1. 信道利用率：有数据通过的时间/（有+无）数据通过的时间
  2. 网络利用率：信道利用率加权平均
    :::tips
    利用率接近于1时时延急剧增大
    :::

计算机网络结构

分层

分层的原因

:::tips
发送文件前要进行的工作

  1. 发起信息通讯的主机必须先把数据通信的通路激活
  2. 告诉网络如何识别目的主机
  3. 发起通信的主机要查明目的主机是否开机，是否与网络连接正常
  4. 发起通讯的计算机要弄清楚，对方计算机中的文件管理程序是否已经做好准备工作
    1. 是否同意接受文件
    2. 是否有空间存储
  5. 确保差错和意外能够被解决
    1. 传送错误
    2. 传重复
    3. 传输丢失
  6. ……..
    :::

分层的原则

:::tips
分层的基本原则

  1. 各层之间相互独立，每层只实现一种相对独立的功能
  2. 镁层之间界面清晰自然，易于理解，相互交流尽可能少
  3. 结构上可分隔开。每层都采用最合适的技术来实现
  4. 报纸下层对上层的独立性，上层单项使用下层提供的服务
  5. 整个分层结构应该能促进标准化工作
    :::

结构图


:::tips

  1. 实体：第n层中的活动元素称为n曾层实体，同一层的实体称为对等实体

  2. 协议：为对等实体数据交换而建立的标准、规则或者约定沉称为网络协议（水平）

    1. 语法：规定传输数据的格式
    2. 语义：规定所要完成的功能
    3. 同步：规定各种操作的顺序

  3. 接口(访问服务点SAP)：上层使用下层服务的入口

  4. 服务：下层为上层提供的功能调用（垂直）
    :::

    :::tips

  5. PCI：协议控制信息：控制协议操作的信息

  6. SDU：服务数据单元：为完成用户操作而传输的信息

  7. PDU：协议数据单元：对等层次之间传输数据的单位
    :::

OSI参考模型（重要）

:::tips
为了支持异构网络系统的互联互通
国际标准化组织（ISO）在1984年提出开放系统互联（OSI）参考模型
:::

发送端每一层都会对信息进行一个包装
接收端会逐层拆掉包装

协议是为了功能服务的

  1. 应用层：用户与网络交互的界面，能和用户产生流量的程序
    1. 协议：
    2. 文件传输（FTP）
    3. 电子邮件（SMTP）
    4. 万维网（HTTP）
    5. ……
  2. 表示层：用于处理在两个通信系统中交换信息的表示方式（语法，语义）
    1. 功能：数据格式转换
    2. 数据加密解密
    3. 数据的压缩和恢复
    4. 协议：JPEG\ASCLL
  3. 会话层：像表示层的实体/用户提供建立连接并在连接上有序地传递数据。会话也是建立同步（SYN）
    1. 功能：建立、管理、终止会话
    2. 使用校验点可以在会话在失效时从校验点/同步点回复通信，实现数据同步。（传输大文件）
    3. 协议：ADSP、ASP
  4. 传输层：负责主机中两个进程之间的通信，端到端的通信传输单位是报文段，用户数据报
    1. 功能：可靠传输、不可靠传输
    2. 差错控制
    3. 流量控制
    4. 复用分用
      1. 复用：多个应用层进程可同时使用下面运输层的服务
      2. 分用：运输层把收到的信息分别交付给上面应用层中相应的进程
    5. 协议：TCP/UDP
  5. 网络层（IP层、网际层）：主要任务是把分组从源端传到目的端，为分组交换网上的不同主机提供服务。网络层传输单位时数据报。
    1. 功能：路由选择（最佳路由）
    2. 流量控制
    3. 差错控制（校验）
    4. 拥塞控制：如果所有节点都来不及接受分组，而要丢弃大量分组，网络就处于拥塞状态。因此要采取一些措施缓解拥塞
    5. 协议：IP、IPX、ICMP、IGMP、ARP、RARP、OSPF
  6. 数据链路层：主要任务时把网络层传下载的数据包组装成帧，传输单位是帧。
    1. 功能成帧（定义帧的开始和结束）
    2. 差错控制 帧错+位错
    3. 流量控制
    4. 访问（接入）控制 控制对信道的访问
    5. 协议：SDLC、HDLC、PPP、STP
  7. 物理层：在物理媒体上实现比特流透明传输 单位是比特
    1. 透明传输：不管传输什么样的比特组合，都能在链路上传送
    2. 功能：定义接口特性
    3. 定义传输模式 单工（只能有一个接收方一个发送方）、半双工（双方都可以发送和接受，但是同一时间只能有一个接收方和发送方）、双工（双方可以同时发送和接受）
    4. 定义传输速率
    5. 比特同步
    6. 比特编码曼彻斯特编码
    7. 协议：Pj45、802.3

物理层

概念

:::tips
概念：物理层指的是如何解决如何在传输媒体上传输比特流，而不是指具体的传输介质。
任务：确定与传输媒体接口有关的特性。—>定义标准
:::
:::tips

  1. 机械特性：定义物理连接特性，规定物理连接时所采用的规格、接口形状、引线数目、引脚数量和排列情况
  2. 电器特性：规定传输二进制位时，线路上信号的电压范围、阻抗匹配、传输速率和距离限制信息等
  3. 功能特性：指明某条线上出现的某一电平表示何种意义，接口部件的信号线的用途
  4. 规程特性（过程特性）：定义各条线路的工作规程和时序关系
    :::

数据通信

数据通讯模型

:::tips

  • 通信的目的：是传送信息，如文字，图像和视频等。
  • 数据是指：传送信息的实体。
  • 信号是指：数据的电气或电磁表现。
    • 数字信号：代表消息的信号是离散的
    • 模拟信号：代表消息的参数是连续的
  • 信源：产生和发送信息的源头
  • 信宿：目的站、接收信息的终点
  • 信道：信号的传输媒介。一班用来表示向某一个方向传送信息的介质，因此一条通信线路往往包含一条发送信道和一条接收信道
    :::

    三种通信方式
  1. 单工通信：只有一个方向上的通信，仅需要一条信道
  2. 半双工通信：通信双方都可以进行发送和接受，但是不能发送的同时接收，需要两条信道
  3. 双工通信：通信双方可以同时发送和接收信息，需要两条信道

两种数据传输方式

  1. 串行传输：速度慢、费用低、适用于远距离传输
  2. 并行传输：速度快、费用高、适合近距离传输（用于计算机内部的传输）


码元、波特、速率、带宽

码元

:::tips
码元是指：用一个固定时长的信号波形(数字脉冲)，代表不同离散数值的基本波形，是数字通信中数字信号的计量单位，这个时长内的信号称为k进制码元，而该时长称为码元宽度。当码元的离散状态有M个时(M大于2)，此时码元为M进制码元。
1码元可以携带多个比特的信息量。
例如，在使用二进制编码时，只有两种不同的码元，一种代表0状态，另一种代表1状态。
:::

速率

:::tips
速率也叫数据率，是指数据的传输速率，表示单位时间内传输的数据量。可以用码元传输速率和信息传输速率表示。

  1. 码元传输速率:别名码元速率、波形速率、调制速率、符号速率等，它表示单位时间内数字通信系统所传输的码元个数(也可称为脉冲个数或信号化的次数)，单位是波特(Baud)。1波特表示数字通信系统每秒传输一个码元。这里的码元可以是多进制的，也可以是二进制的，但码元速率与进制数无关。
    1. 相当于一秒传输多少码元
  2. 信息传输速率:别名信息速率、比特率等，表示单位时间内数字通信系统传输的二进制码元个数(即比特数)单位是比特/秒 (b/s)
    1. 相当于一秒传输多少比特

关系:若一个码元携带n bit的信息量，则M Baud的码元传输速率所对应的信息传输速率为MXn bit/s
:::

带宽

:::tips
带宽:表示在单位时间内从网络中的某一点到另一点所能通过的“最高数据率”，常用来表示网络的通信线路所能传输数据的能力。单位是b/s。
:::

奈氏准则和香农定理

失真


:::tips
影响是失真的程度

  1. 码元传输速率（越快失真度越大）‘
  2. 信号传输距离（距离越远受到干扰的时间越久）
  3. 噪声干扰（越多越容易失真）
  4. 传输媒体的质量（质量越差越容易失真）
    :::

码间串扰

:::tips
码间串扰：接收端收到的信号波形失去了码元之间清晰的界限导致不能区分的现象。（码元间的干扰）
:::
信道带宽是信道能通过最高频和最低频之差
200HZ不能通过的原因是频率太低，到达另一端时信号已经损耗没了。
4000HZ不能通过的原因是接收端不能区分高电平和低电平之间的差异（码间串扰）

奈氏准则

:::tips
奈氏准则：在理想低通(无噪声，带宽受限)条件下，为了避免码间串扰，极限码元传输速率为 2W Baud，W是信道带宽，单位是Hz。
:::

:::tips

  1. 在任何信道中，码元传输的速率是有上限的。若传输速率超过此上限，就会出现严重的码间串扰问题，使接收端对码元的完全正确识别成为不可能。
  2. 信道的频带越宽(即能通过的信号高频分量越多)，就可以用更高的速率进行码元的有效传输
  3. 奈氏准则给出了码元传输速率的限制，但并没有对信息传输速率给出限制。
  4. 由于码元的传输速率受奈氏准则的制约，所以要提高数据的传输速率，就必须设法使每个码元能携带更多个比特的信息量，这就需要采用多元制的调制方法
    :::

香农定理

:::tips
香农定理：在带宽受限且有噪声的信道中，为了不产生误差，信息的数据传输速率有上限值。
噪声存在于所有的电子设备和通信信道中。由于噪声随机产生，它的瞬时值有时会很大，因此噪声会使接收端对码元的判决产生错误。但是噪声的影响是相对的，若信号较强，那么噪声影响相对较小。因此，信噪比就很重要。
:::


:::tips

  1. 信道的带宽或信道中的信噪比越大，则信息的极限传输速率就越高
  2. 对一定的传输带宽和一定的信噪比，信息传输速率的上限就确定了。
  3. 只要信息的传输速率低于信道的极限传输速率，就一定能找到某种方法来实现无差错的传输
  4. 香农定理得出的为极限信息传输速率，实际信道能达到的传输速率要比它低不少。
  5. 从香农定理可以看出，若信道带宽W或信噪比S/N没有上限(不可能)，那么信道的极限信息传输速率也就没有上限。
    :::

编码和调制

:::tips
信道上传送的信号有两种：

  1. 基带信号：将数字信号1和0直接用两种不同的电压表示，再送到数字信道上去传输(基带传输)。来自信源的信号，像计算机输出的代表各种文字或图像文件的数据信号都属于基带信号。基带信号就是发出的直接表达了要传输的信息的信号，比如我们说话的声波就是基带信号。
  2. 宽带信号：将基带信号进行调制后形成的频分复用模拟信号，再传送到模拟信道上去传输(宽带传输)。把基带信号经过载波调制后，把信号的频率范围搬移到较高的频段以便在信道中传输(即仅在一段频率范围内能够通过信道)。
    :::
    :::tips
    在传输距离较近时，计算机网络采用基带传输方式(近距离衰减小，从而信号内容不易发生变化)
    在传输距离较远时，计算机网络采用宽带传输方式(远距离衰减大，即使信号变化大也能最后过滤出来基带信号
    :::
    :::info
    把数据转化成数字信号的是编码。
    把数据转化成模拟信号的是调制。
    :::

    数字数据（数字发送器） 数字信号 编码
    数字数据（调制器） 模拟信号 调制

模拟数据（PCM编码器）数字信号 编码
模拟数据（放大器调制器）模拟信号 调制

编码方式

数字数据编码为数字信号有以下几种编码方式

  1. 非归零编码（NRZ）
  2. 曼彻斯特编码
  3. 差分曼彻斯特编码
  4. 归零编码（RZ）
  5. 反向不归零编码（NRZI）
  6. 4B/5B编码


非归零编码

编码容易实现，但是没有检错功能，并且无法判断一个码元的开始和结束，以至于收发双方难以保持同步。
当使用非归零编码发送数据，如果发送一长条“1”的话那么信号就会是一个长条，难以知道是多少个1，是所以为了解决，那么就需要在建立一条信道，告诉接收端我的一个时钟周期是多少（多久是一个bit）

归零编码


信号在一个码元内都要归为零的编码方式
信道处于低电平的时间较多，相当于信贷没怎么使用，不推荐使用，利用率低。

反向不归零编码


信号电平反转表示0，信号点评不变表示1.
如果发送端发送数据全为0时，那么就非常好接受，但是如果全为1时那么覅那位不发生跳变，很难进行接收。还是要建立一个新的信道，规定一个时钟周期有多长。

曼彻斯特编码

:::tips
优秀的编码，不需要别的信道进行同步，自同步，根据信号就能判断出来数据是什么，不用计算时钟周期/
将一个码元分成两个时间相等的间隔，前一个间隔为低电平后一个间隔为高电平表示码元1，码元0正好相反。也可以使用完全相反的规定。
编码的特点是每一个码元内进行一次电平跳变，中间的信号既可以作为时钟信号（用于同步），又可以作为数据信号，但是它所占用的带宽长度就是原始基带带宽的两倍。
:::

差分曼彻斯特编码

:::tips
常用于局域网的传输 同1异0
若为1那么前一个码元的后半段与后一个码元的前半段电平相同，若为0则相反。
自同步
抗干扰性强于曼彻斯特编码。
看开始边界有没有跳变
:::

4B/5B编码

:::tips
bit流中插入额外的比特来打破一连串的1或者0，就是用五个bit来编码4个bit的数据，再传给接收方传输效率只有80%。
:::

调制方式

数字数据调制技术在发送端将数字信号转换为模拟信号，而在接收端将模拟信号还原为数字信号，分别对应于调制解调器的调制和解调过程。

  1. 调幅（2ASK）
  2. 调频（2FSK）
  3. 调相（2PSK）
  4. 调幅+调相（QAM）


解调

  1. 抽样：对模拟信号进行扫描，把时间连续的信号变成离散的数字信号
    :::info
    为了抽样取得的离散信号能够无失真的代表被抽样的模拟数据需要采样定理进行采样。
    采样频率>=2*最高信号频率
    :::

  2. 量化：把抽样取得的电平幅值按照一定的分级度，转化为对应的数字值。并取整数。这就把连续的电平幅值，转化为离散的数字信号。

  3. 把量化的结果转化为二进制编码

传输介质

:::tips
传输介质也称传输媒体/传输媒介，它就是数据传输系统中在发送设备和接收设备之间的物理通路。
传输媒体并不是物理层。传输媒体在物理层的下面，因为物理层是体系结构的第一层，因此有时称传输媒体为0层。在传输媒体中传输的是信号，但传输媒体并不知道所传输的信号代表什么意思但物理层规定了电气特性，因此能够识别所传送的比特流。
:::
传输介质：

  1. 导向性传输介质：电磁波被导向沿着固体媒介（铜丝，光纤）传播。
  2. 非导向型传输介质：自由空间，介质可以是空气、真空、海水等。

导向性传输介质

双绞线


双绞线是是古老又常用的传输介质、是由两根采用一定规则并排铰合的、相互绝缘的铜导线组成。
绞合可以减少对相邻导线的电磁干扰。
:::tips
双绞线价格便宜，是最常用的传输介质之一，在局域网和传统电话网中普遍使用。模拟传输和数字传输都可以使用双绞线，其通信距离一般为几公里到数十公里。距离太远时，对于模拟传输，要用放大器放大衰减的信号;对于数字传输，要用中继器将失真的信号整形。
:::

同轴电缆

:::tips
同轴电缆由导体铜质芯线、绝缘层、网状编织屏蔽层和塑料外层构成。按特性阻抗数值的不同，通常将同轴电缆分为两类:50Q同轴电缆和750同轴电缆。

  1. 50Q同轴电缆主要用于传送基带数字信号，又称为基带同轴电缆，它在局域网中得到广泛应用
  2. 750同轴电缆主要用于传送宽带信号，又称为宽带同轴电缆，它主要用于有线电视系统
    :::

光纤


:::tips
光纤通信就是利用光导纤维(简称光纤)传递光脉冲来进行通信。有光脉冲表示1，无光脉冲表示0。而可见光的频率大约是108MHz，因此光纤通信系统的带宽远远大于目前其他各种传输媒体的带宽。
:::
:::tips
光纤主要由纤芯(实心的!)和包层构成，光波通过纤芯进行传导。
包层较纤芯有较低的折射率。当光线从高折射率的介质射向低折射率的介质时，其折射角将大于入射角。因此，如果入射角足够大，就会出现全反射，即光线碰到包层时候就会折射回纤芯、这个过程不断重复，光也就沿着光纤传输下去。
:::

  1. 多模光纤
    1. 纤芯可以传输多条光线
  2. 单模光纤
    1. 纤芯非常细，只传递一条光波，相当于直接射到接收端，损耗很小

特点：

  1. 传输损耗小，中继器距离长，对远距离传输十分经济
  2. 抗雷电，抗电磁干扰好。
  3. 无串音干扰，保密性好，不易被窃听或者截取数据
  4. 体积小，重量轻

非导向性传输介质

  1. 无线电波：向所有方向传播、
    :::tips
    穿透能力较强，可以传递远距离，多用于通信领域
    :::

  2. 微波：信号沿着固定方向传播
    :::tips
    通信频率高，频段范围宽，数据率很高

  3. 地面微波接力通信

  4. 卫星通信

    1. 通信容量大
    2. 距离远
    3. 覆盖广
    4. 广播通信、多地址通信
    5. 缺点：传播延时长、受气候影响、误码率高、成本高
      :::

  5. 红外线、激光：信号沿着固定方向传播
    :::tips
    把要传输的信号转化为各自的信号形式、红外光信号、激光信号、再向空中传播。
    :::

物理层设备

中继器


:::tips
原因：由于距离较远，信号传输有损失，衰减到一定程度就会无造成失真，因此导致接收错误。
作用：对信号进行再生和还原，对衰减的信号进行放大，保持原数据相同，增大传输距离延长网络长度。
:::
:::tips
中继器的两端

  1. 是网段，而不是子网，适用于两个相同的两类网络连接，两个网段速率要相同。
  2. 中继器只将任何电缆段上的数据发送到另一段电缆上，它仅作用于信号的电气部分，并不管数据中是否有错误数据或不适于网段的数据。
  3. 两端可以连相同媒体，可以连不同媒体
  4. 中继器两端一定是同一个协议
    :::
    :::tips
    5-4-3规则：网络标准中都对信号的延迟范围作了具体的规定，因而中继器只能在规定的范用内进行，否则会网络故障。
    :::

集线器（多口中继器）


:::tips
集线器的功能：对信号进行再生放大转发，对衰减的信号进行放大，接着转发到其他所有(除输入端口外)处于工作状态的端口上，以增加信号传输的距离，延长网络的长度。不具备信号的定向传送能力，是一个共享式设备
:::

数据链路层

:::tips
基本概念

  1. 节点：主机、路由器

  2. 链路：网络中两个节点之间的物理通道，链路的传输介质分为有线链路、无线链路。

  3. 数据链路：网络中两个节点的逻辑通道，把实现控制数据传输协议的硬件和软件加到链路上就构成数据链路。

  4. 帧：链路层的协议数据单元、封装网络层的数据报。
    :::
    :::tips
    主要功能：
    数据链路层在物理层提供服务的基础上向网络层提供服务，其最基本的服务是将源自网络层来的数据可靠地传输到相邻节点的目标机网络层。其主要作用是加强物理层传输原始比特流的功能，将物理层提供的可能出错的物理连接改造成为逻辑上无差错的数据链路，使之对网络层表现为一条无差错的链路。

  5. 为网络层提供服务，无确认无连接服务，有确认无连接服务，有确认面向连接服务

  6. 链路管理：连接的建立，维持，释放。
    :::
