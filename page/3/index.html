Redis笔记 2024/3/28

1. Redis Sorted Set

介绍

Redis 有序集合和集合一样也是 string 类型元素的集合,且不允许重复的成员。

不同的是每个元素都会关联一个 double 类型的分数。redis 正是通过分数来为集合中的成员进行从小到大的排序。

有序集合的成员是唯一的,但分数(score)却可以重复。

集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是 O(1)。 集合中最大的成员数为 2^32^ - 1 (4294967295, 每个集合可存储40多亿个成员)。

命令

名称 描述
zadd [key] [score1] [member1] [score2] [member2] … 向有序集合添加一个或多个成员，或者更新已存在成员的分数
zcard [key] 获取该有序集合的成员数
zcount [key] [min] [max] 获取指定分数区间的成员个数
zincrby [key] [increment] [member] 对指定成员的分数加上增量increment
zrangebyscore [key] [min] [max] 返回有序集合中指定分数区间的成员列表，有序集成员按分数值递增(从小到大)次序排列，加上 withscores可以连同分数一起返回
zrange [key] [min] [max] 返回指定区间内的成员列表，加上 withscores可以连同分数一起返回
zrevrangebyscore key maxmin [withscores] [limit offset count] 同zrangebyscore，只是倒序（从大到小）
zrem [key] [member] 删除指定的成员
zrank [key] [member] 返回该成员在集合中的排名，排名在这里是从0开始的

数据结构

SortedSet(zset)是Redis提供的一个非常特别的数据结构，一方面它等价于Java的数据结构Map<String, Double>，可以给每一个元素value赋予一个权重score，另一方面它又类似于TreeSet，内部的元素会按照权重score进行排序，可以得到每个元素的名次，还可以通过score的范围来获取元素的列表。

==zset底层使用了两个数据结构：==

  1. hash，hash的作用就是关联元素value和权重score，保障元素value的唯一性，可以通过元素value找到相应的score值
  2. 跳跃表，跳跃表的目的在于给元素value排序，根据score的范围获取元素列表

案例

有序集合在生活中比较常见，例如根据成绩对学生排名，根据得分对玩家排名等。对于有序集合的底层实现，可以用数组、平衡树、链表等。数组不便元素的插入、删除；平衡树或红黑树虽然效率高但结构复杂；链表查询需要遍历所有效率低。Redis采用的是跳跃表。跳跃表效率堪比红黑树，实现远比红黑树简单。

  1. 跳跃表

    要查找值为51的元素，需要从第一个元素开始依次查找、比较才能找到。共需要6次比较。

  2. 跳跃表

    从第2层开始，1节点比51节点小，向后比较。

    21节点比51节点小，继续向后比较，后面就是NULL了，所以从21节点向下到第1层

    在第1层，41节点比51节点小，继续向后，61节点比51节点大，所以从41向下

    在第0层，51节点为要查找的节点，节点被找到，共查找4次。

Redis Bitmaps

介绍

现代计算机用二进制（位） 作为信息的基础单位， 1个字节等于8位， 例如“abc”字符串是由3个字节组成， 但实际在计算机存储时将其用二进制表示， “abc”分别对应的ASCII码分别是97、 98、 99， 对应的二进制分别是01100001、 01100010和01100011，如下图

合理地使用操作位能够有效地提高内存使用率和开发效率。

（1）Bitmaps本身不是一种数据类型， 实际上它就是字符串（key-value） ， 但是它可以对字符串的位进行操作。

（2）Bitmaps单独提供了一套命令， 所以在Redis中使用Bitmaps和使用字符串的方法不太相同。 可以把Bitmaps想象成一个以位为单位的数组， 数组的每个单元只能存储0和1， 数组的下标在Bitmaps中叫做偏移量。

命令

名称 描述
setbit [key] [offset] [value] 设置Bitmaps中某个位置的值
getbit [key] [offset] 获取Bitmaps中某个位置的值
bitcount [key] [start] [end] 统计字符串被设置为1的bit数，一般情况下，给定的整个字符串都会被进行计数，通过指定额外的 start 或 end 参数，可以让计数只在特定的范围内进行
bitop [and/or/not/xor] [newkey] [key…] bitop是一个复合操作， 它可以做多个Bitmaps的and（交集） 、 or（并集） 、 not（非） 、 xor（异或） 操作并将结果保存在destkey中

Bitmaps和set对比

假设网站有1亿用户， 每天独立访问的用户有5千万， 如果每天用集合类型和Bitmaps分别存储活跃用户可以得到表

set和Bitmaps存储一天活跃用户对比<

数据类型 每个用户id占用空间 需要存储的用户量 全部内存量
集合类型 64位 50000000 64位*50000000 = 400MB
Bitmaps 1位 100000000 1位*100000000 = 12.5MB

很明显， 这种情况下使用Bitmaps能节省很多的内存空间， 尤其是随着时间推移节省的内存还是非常可观的

set和Bitmaps存储独立用户空间对比

Bitmaps 12.5MB 375MB 4.5GB

但Bitmaps并不是万金油， 假如该网站每天的独立访问用户很少， 例如只有10万（大量的僵尸用户） ， 那么两者的对比如下表所示， 很显然， 这时候使用Bitmaps就不太合适了， 因为基本上大部分位都是0

Redis String

简介

  1. String是Redis最基本的类型，一个key对应一个value
  2. String类型是 二进制安全 的，意味着Redis的 string 可以包含任何类型的数据，比如jpg图片或者序列化的对象
  3. String类型是Redis最基本的数据类型，一个value对应的字符串的大小最多可以是 512M

常用命令

  • set [key] [value]
  • get [key] 查看key对应的value
  • append [key] [value] 将指定的value追加到key所对应的原字符串中，并返回新字符串的长度（如果该键不存在，则创建）
  • strlen [key] 获得字符串的长度
  • setnx [key] [value] 只有在key不存在时，才能设置value（即：不能进行值覆盖的set命令）


  • incr [key] 将key中储存的数字加1（只能对数字值进行操作，如果为空，新增值为1的字符串）
  • decr [key] 将key中存储的数字减1
  • incrby / decrby [key] [length] 将key中存储的数字按指定的步长进行增减（自定义步长，如果为空，新增值为步长的字符串）


  • mset [key1] [value1] [key2] [value2] [key3] [value3] … 同时设置多个key
  • mget [key1] [key2] … 同时获得多个值
  • msetnx [key1] [value1] [key2] [value2] … 当且仅当所有给定的key都不存在时，可以使用
  • getrange [key] [startIndex] [endIndex] 获得某个范围内的字符串切片，类似java的 substring，它是左右下标都要包含的
  • setrange [key] [startIndex] [value] 用value覆盖某个下标对应的值（索引从0开始）
  • setex [key] [expireTime] [value] 设置键值对的同时设置过期时间(expire time)
  • getset [key] [value] 类似于set的覆盖，只不过它是设置一个新value，同时返回一个老value

数据结构

String的数据结构为简单动态字符串(Simple Dynamic String,缩写SDS)。是可以修改的字符串，内部结构实现上类似于Java的ArrayList，采用预分配冗余空间的方式来减少内存的频繁分配。

如图中所示，内部为当前字符串实际分配的空间capacity一般要高于实际字符串长度len。当字符串长度小于1M时，扩容都是加倍现有的空间，如果超过1M，扩容时一次只会多扩1M的空间。需要注意的是字符串最大长度为512M。

Redis笔记 2024/3/28

Redis List

简介

  1. Redis List是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部或者尾部
  2. 它的底层实际是个 双向链表 ，对两端的操作性能很高，通过索引下标的操作中间的节点性能会比较差

常用命令

  • lpush / rpush [key] [value1] [value2] [value3] … 从左边 / 右边插入一个或多个值
  • lpop / rpop [key] 从左边 / 右边吐出一个值（吐出以后，值就从该列表消失了）
  • rpoplpush [sourceList] [destination] 从sourceList中右推出一个值，然后左push到destination中
  • lrange [list] [start] [end] 按照索引下标访问指定的值（-1表示右边第一个，从0到-1表示访问所有值）
  • lindex [list] [index] 访问指定下标的值
  • llen [list] 获得列表长度
  • linset [list] before / after [value] [newvalue] 在value值的前面 / 后面插入一个newvalue
  • lrem [list] [number] [value] 从左边开始，删除number个值为[value]的元素
  • lset [list] [index] [value] 将列表指定索引的值进行替换

Redis List 数据结构

List的数据结构是 快速链表 quickList

首先，列表元素较少的情况下会使用一块连续的内存，这个结构是 ziplist ，即：压缩列表

它将所有的元素紧挨着一起存储，分配的是一块连续的内存，当数据量比较多的时候才会改成 quickList

当数据量比较大的时候才会改成 quickList

因为普通的链表需要的附加指针空间太大，会比较浪费空间。比如这个列表里存的只是int类型的数据，结构上还需要两个额外的指针 prev 和 next

Redis 将链表和 ziplist 结合起来组成了 quicklist ，也就是将多个 ziplist 使用双向指针串起来使用。这样既满足了快速的插入删除性能，又不会出现太大的空间冗余

  1. Redis 发布订阅 (pub/sub) 是一种消息通信模式：发送者 (pub) 发送消息，订阅者 (sub) 接收消息

    Redis 客户端可以订阅任意数量的频道。

  2. 客户端可以订阅频道如下图

    当给这个频道发布消息后，消息就会发送给订阅的客户端

实现

  • 打开一个Redis客户端，订阅频道 channel1

  • 打开另一个Redis客户端，向频道 channel1 中发送消息 "hello"

Redis Hash

介绍

Redis hash 是一个 键值对集合 。

Redis hash 是一个 string 类型的 field 和 value 的映射表，hash特别适合用于存储对象。

类似Java里面的 Map<String,Object>

==示例：==

用户ID为查找的key，存储的value用户对象包含姓名，年龄，生日等信息，如果用普通的key/value结构来存储

主要有以下2种存储方式：

命令

名称 描述
hset [key] [filed] [value] 给key集合中的 field 赋值 value
hget [key] [field] 获得key集合中的 field 的值
hmset [key] [field1] [value1] [field2] [value2] … 批量设置 filed 的 value
hmget [key] [field1] [field2] … 批量获取 field 的 value（此操作不是原子性的，不会因为其中一个field获取不到就返回错误，获取不到会为该值返回 nil）
hexists [key] [field] 查看哈希表key中，filed 是否存在（存在返回1，不存在返回0）
hkeys [key] 列出该哈希表中所有的 field
hvals [key] 列出该hash集合中的所有 value
hgetall [key] 获取该哈希表中所有的 field 和 value
hlen [key] 获取哈希表中字段的数量
hincrby [key] [field] increment(可选，默认为1) 为哈希表中某一 field 的 value 进行 +increment（返回加后的结果）
hsetnx [key] [field] [value] 当切仅当 field 不存在时，设置其值为 value（不成功返回0，成功返回1）

数据结构

Hash类型对应的数据结构是两种：

  1. ziplist（压缩列表）
  2. hashtable（哈希表）

当field-value长度较短且个数较少时，使用ziplist，否则使用hashtable

Jedis

所需的jar包

        <!--Jedis-->
        <dependency>
            <groupId>redis.clients</groupId>
            <artifactId>jedis</artifactId>
            <version>3.2.0</version>
        </dependency>

连接之前必须做的事情

  1. redis.conf 中注释掉 bind 127.0.0.1
  2. 将 redis.conf 中的 protected mode yes 改为 no
  3. 禁用Linux的防火墙：Linux(CentOS7)里执行命令 systemctl stop firewalld

创建测试类

public class JedisDemo1 {

    public static void main(String[] args) {
        //创建Jedis对象
        Jedis jedis = new Jedis("192.168.0.116", 6379);//arg1:ip arg2:端口号
    
        String ping = jedis.ping();//需提前关掉linux内的防火墙，ping通的话就返回一个 "PONG"
        System.out.println(ping);
    }
}

==成功返回：”PONG”==

         * @description:
